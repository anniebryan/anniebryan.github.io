Getting Started

A bit about myself, this course, and how it can help you.

About Me

Hi! My name is Annie Bryan, and I'm a software engineer at Meta (formerly Facebook). I just graduated from MIT with a double major in Computer Science and Mathematical Economics.

My sophomore year of college, I spent a lot of time applying to SWE internships and preparing for interviews. It was a stressful process to say the least, and many interviews did not go as well as they could have.

After dozens of technical interviews and online assessments, I've developed a comprehensive guide to everything I used while I was studying for these interviews. This preparation led to offers from Facebook and a few other big tech companies, and I evem passed the technical round of the Google application process.

Additionally, I've tutored several MIT undergraduate students in data structures, algorithms, and interview prep.

What is this course?

This course is a guide to everything I used when I was studying for technical interviews. It covers the bulk of the technical material that would be reasonably expected of someone applying to a software engineering role.

This course includes the theory as well as practical examples and walkthroughs of practice problems. It's important to know the background behind why something works, but it's also important to be able to apply the concept!

What will this course teach me?

This course will teach you all you need to know to nail your next coding interview and land a job as a SWE for a FAANG company. Here are some examples of what you'll learn.

Complexity

After you write any algorithm or data structure, you'll likely be asked about its time and space complexity.

This section will cover:

  • Big-O notation
  • The runtime of common algorithms
  • Determining the time and space complexity of any algorithms or data structures you develop
  • Knowing if a runtime is amortized, expected-case, or worst-case

Additionally, this section will cover some more advanced topics that are less likely to show up in an interview, but are nonetheless important fundamentals to know as a SWE:

  • Complexity classes such as P, NP, NP-Hard, and NP-Complete
  • Reductions

Sorting

Coding interviews often require a data structure to be sorted, so it's crucial to know the runtime of a few sorting algorithms, and to be able to implement them. This section will cover the following sorting algorithms:

  • Merge sort
  • Quick sort
  • Radix sort
  • Insertion sort
  • Heap sort

Data Structures

Data structures are the bread and butter of coding interviews. It's important to be familiar with these, as optimized solutions to many advanced questions will involve one or more. This section will cover the following data structures:

  • Arrays
  • Sets
  • Hash Tables
  • Stacks and Queues
  • Linked Lists
  • Trees, AVLs, Tries, Heaps, etc.

Graph Algorithms

Lots of problems can be abstracted by converting it to a graph and applying well-known graph algorithms such as those to find a path of minimum distance, determine connectivity, or detect cycles.

This section will cover the pros and cons of the different ways to represent a graph in memory:

  • Object and pointers
  • Adjacency matrix
  • Edge list
  • Adjacency map

the basic graph traversal algorithms:

  • Breadth-first search (BFS)
  • Depth-first search (DFS)

some common algorithms for single-source shortest path (SSSP):

  • Directed acyclic graph shortest path (DAGSP)
  • Dijkstra's
  • Bellman-Ford

algorithms for determining connectivity, detecting cycles, and other useful graph problem strategies.

Dynamic Programming

Many advanced interview questions can be solved recursively using dynamic programming. This section will cover the 5 steps I learned in my college algorithms course:

  1. Subproblem
  2. Relation
  3. Base case
  4. Solution
  5. Time (run time)

The best method of getting comfortable with DP problems is to practice, so this section also includes several practice problems with example solutions.

Who should use this course?

This course is targeted to anyone applying for a software engineering (SWE) internship or full-time position. This course will be most useful to you if you already know Python or JavaScript, as my tutorials and practice examples will be written in one or both of those languages.
